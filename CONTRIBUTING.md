# Contributing to rapidapi-setup
Thank you for considering contributing to rapidapi-setup! We appreciate your help in making this project better.

## Forking the Repository
To contribute to rapidapi-setup, follow these steps to fork the repository:
1. Go to the [rapidapi-setup repository](https://github.com/rapidapi/rapidapi-setup) on GitHub.
2. Click the "Fork" button in the top-right corner of the page.
3. Choose the account where you want to fork the repository.
4. Wait for the fork to complete.

## Branch Naming
When creating a new branch, please use the following naming convention:
`type/description`
Where `type` is one of:
* `feat` for new features
* `fix` for bug fixes
* `docs` for documentation changes
* `style` for code style changes
* `refactor` for code refactoring
* `test` for test additions or changes
* `chore` for miscellaneous changes
And `description` is a brief summary of the changes.

## Commit Style
We use the conventional commit style:
`type(scope): brief summary`
Where `type` is the same as in the branch naming convention, `scope` is optional, and `brief summary` is a short description of the changes.
For example:
`feat(api): add new endpoint`

## Pull Request Template
When submitting a pull request, please use the following template:
### Description
* Briefly describe the changes made
### Changes
* List the changes made, using bullet points if necessary
### Related Issues
* Link to any related issues or pull requests
### Checklist
* [ ] I have tested the changes
* [ ] I have updated the documentation (if necessary)
* [ ] I have followed the code style guidelines
